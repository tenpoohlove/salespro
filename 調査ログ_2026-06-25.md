# P3（Pitch Navi）不具合 — 調査と修正の全記録
最終更新: 2026-06-25

## 症状
- CursorのClaude CodeでP3だけ、「続きをやりたい」直後に20〜25分思考が止まらない。
- 返答が噛み合わない。実際には存在しない指示を見たと思い込んで報告する（作話）。
- 「push済み」を「未push」と言うなど、確認していない事実を断定してしまう。
- 同じ環境のP4は正常。ウィンドウを閉じてもPC再起動しても直らない。
- 3日前(6/22)のサービス障害の日から、P3だけずっとこの状態。

---

## 調査の流れ（仮説 → 検証 → 結果）

### 仮説1: 起動時に読むファイルに、想定外の文字列が紛れている？
- 検証: P3プロジェクト全ファイルを検索（log/bak/state含む）
- 結果: **該当なし**。AIが「見た」と言った指示はファイルに実在せず、作話（思い込み）と判明。

### 仮説2: 起動時に走るフックが余計な処理をしている？
- 検証: P3の`.claude/`、Windows側ユーザー全体の設定(settings.json)を確認
- 結果: フックは「終わったらビープ音を鳴らす」だけ。該当なし。

### 仮説3: 「続きをやりたい」で巨大な過去会話を読み込んでいる？
- 検証: セッション履歴(`~/.claude/projects/...`)のサイズと中身を確認
- 結果: CTXは6〜7%と低く、巨大履歴は読んでいない。むしろP4の方が履歴大。＝量の問題ではない。

### 仮説4: 壊れた「会話の要約(compactサマリー)」が残っている？
- 検証: 全セッションでcompact要約の有無を確認
- 結果: 要約は1件も存在しない。＝否定。

### 仮説5: 思考量を最大にする設定(MAX_THINKING_TOKENS)やキーワードがある？
- 検証: 設定ファイル・環境変数・キーワードを確認
- 結果: 設定なし。該当キーワードもP3には無い。＝否定。

### 決定的検証: 思考量を時系列で定量化（セッション記録の思考署名サイズを測定）
```
6/12〜6/20（障害前）… 最大 6〜34KB（正常）
6/22 午後（障害日） … 227KB・大きな思考8回 ← ここで急増
6/24:252KB / 6/25:228KB … 以後ずっと継続
P4（正常）は同日も最大4KB
```
→ 同じOpus 4.8・同mode・設定もキーワードも無いのに、**P3だけ思考量50倍**。かつ「6/22の障害の日ちょうど」に始まっている。

## 突き止めた根本原因

不具合でも容量でも外部要因でもない。原因は2つの「混乱の素」：

### (1) 食い違いだらけの引き継ぎノート
- HANDOFF.md(275行) と 毎回読み込むメモリ(113行) に、過去5回分の食い違う「本番デプロイ済み(HEAD=efc57b3 / 9256a25 / 69d954d …)」が消されずに積み重なっていた。
- AIは「結局いま何が本番か」を確定できず、食い違いを解こうとして長時間の推論に入った。

### (2) 改行コードによる「見かけ上の大量変更」
- `.gitattributes` が無く、OneDrive↔WSLの往復でファイルの改行コード(CRLF↔LF)だけが全行書き換わっていた。
- 中身は1文字も変わっていないのに、git上は「index.htmlが4868行変更」等と表示。AIが「大量の未保存作業がある」と誤認した。

### (3) なぜP3だけ・なぜ再起動でも直らないか
- 6/22のサービス障害が「P3の作業の途中」で中断させ、その中途半端な状態がディスクに残った。P4は中断されず正常。
- 食い違いはディスク上のファイルに残っているので、再起動では消えない。

---

## 実施した修正

### B-1: 混乱の素を取り除く整理 ※機能コードは1行も変更なし
1. `.gitattributes` を追加（`* text=auto eol=lf`）→ 改行差分を恒久的に解消。git status が常にクリーンに。
2. HANDOFF.md を 275行 → 78行に圧縮 → 食い違う過去デプロイ報告を除去。「確定状態＋次の一手」だけに。※過去ログはgit履歴に全部残存。
3. 毎回読み込むメモリ(project_next_steps) を圧縮（元は `.bak-20260625` 保存）
4. debug.log / *.bak を `.gitignore`
5. 今回の調査結果をメモリに記録
- → ローカルコミット `eea2698`

### B-2: 本番反映
6. `git push origin main`（eea2698 + 最新 bd8db77 をGitHubへ）
7. VM(pitch-navi-vm)で再ビルド（git reset→build→声フラグtrue→再up）
8. 検証で動作確認

## 今日の教訓
