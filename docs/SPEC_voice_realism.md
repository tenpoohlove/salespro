## 0. 背景（なぜやるか）

現状のお手本音声には2つの問題がある。
1. **動線が悪い**：声サンプル＋同意の設定UIが評価結果の「下」にあるため、スクロール中にお手本ボタンを押してしまい、下まで行って設定→また戻る羽目になる。
2. **お手本が淡々**：今のお手本は短く、間も抑揚もなく一本調子で読み上げるだけ。これでは「クロージングのお手本」にならない。

これを「①声設定を評価の上へ移動」「②間・抑揚・対話を反映した本物のお手本音声」に直す。

## 1. ゴール（受け入れ基準）

- [ ] 声サンプル＋同意の `#voicePanel` が、評価結果（`#output`）より**上**に表示される。分析開始時から見える。
- [ ] お手本音声に**間（沈黙）・抑揚・緩急・対話のテンポ**が入っていて、聞いて「お手本らしい」と分かる。
- [ ] お手本は**2モードを選べる**：
  - **対話版**：営業（本人クローン声）＋客（汎用声）の掛け合い。
  - **語り版**：営業の語りのみ。客の相づち部分は「想定した間（沈黙）」で表現。
- [ ] 既存機能（分析・フル尺・キャッシュ・BYOK）を壊さない。`FEATURE_VOICE_CLONE` フラグ配下のまま。
- [ ] `npm test` と `npx tsc --noEmit` が通る。ローカルで実際に音声を生成して間・抑揚を耳で確認済み。

## 2. 確定事実：Fish Audio TTS の正しい使い方（最新ドキュメントで確認済み 2026-06-26）

**やってはいけないこと（重要）**
- ❌ **`emotion` パラメータは Fish Audio に存在しない。**絶対に追加しない（過去のコード調査メモに「emotion を追加」という誤った提案があったが誤り）。
- ❌ SSML / `<break time="2s"/>` は**非対応**。使わない。

**間・抑揚を出す正しい手段（この5つを組み合わせる）**
1. **モデル指定**：リクエストヘッダーに `model: s2.1-pro` を付ける（現状は未指定＝旧モデルの可能性。表現力が最も高いのは s2.1-pro）。
2. **インラインタグ**（テキスト内に直接書く・日本語でOK）：
   - `[pause]` … 短い間 / `[long pause]` … 長い間
   - `[ため息]` `[低い声で]` `[強調して]` `[落ち着いた声で]` `[自信を持って]` など自然言語の描写タグも S2.1-Pro は解釈する
3. **温度を上げる**：`temperature` を既定 0.7 → **0.88**、`top_p` 0.7 → **0.85**（抑揚の多様性が増す）。
4. **話速を落とす**：`prosody: { speed: 0.92 }`（落ち着いた営業トーン。0.5〜2.0の範囲）。
5. **セグメント分割＋無音挿入で“正確な秒数の沈黙”を作る**（最重要）：
   - インラインタグの間は秒数が不正確。「クロージング質問の後に2秒」のような**正確な間は、セリフを分割して個別生成し、その間にffmpegで無音を挟んで連結**する。
   - P3は既に `synthesizeDialogue()`＋`concatAudio()`（ffmpeg連結）を持っているので、ここに**無音セグメント挿入**を足すだけで実現できる。

**API呼び出しの形（参考・現コードに合わせて調整）**
```
POST https://api.fish.audio/v1/tts
headers: { Authorization: Bearer <key>, model: "s2.1-pro" }
body: {
  text: "<台本（インラインタグ入り）>",
  reference_id: "<営業=本人voiceId / 客ターンは汎用声or空>",
  temperature: 0.88,
  top_p: 0.85,
  prosody: { speed: 0.92 },
  format: "mp3",
  mp3_bitrate: 128,
  normalize: true
}
```

## 3. 確定事実：クロージングの喋り方（台本に埋め込むデリバリー指示）

台本生成プロンプトに、この「デリバリー早見表」を反映させる。お手本台本は下記ルールで作らせる。

| 場面 | 入れる指示 | 実装での表現 |
|---|---|---|
| クロージング質問の直後 | 1.5〜3秒の完全沈黙 | `[[SILENCE:2000]]`（後述の独自記法） |
| 価格・金額提示の直後 | 3〜5秒の沈黙 | `[[SILENCE:3500]]` |
| 重要キーワードの直前 | 0.5〜2秒の間 | `[pause]` または `[[SILENCE:800]]` |
| 言い切り断言 | 語尾を下げる・低く・ゆっくり | `[自信を持って]` ＋ 文末は言い切り（「〜です。」「進めましょう。」） |
| 共感フレーズ | 柔らかく・ゆっくり・低め | `[落ち着いた声で]` |
| 最終クロージング | 低く・遅く（Sincerityトーン） | `[低い声で、ゆっくり]` |
| 選択型質問 | 語尾を上げてからすぐ下げる | 文を「AとB、どちらが合っていそうですか？」型に |
| 異議の受け止め | 1〜2秒の間＋相手の言葉を繰り返す | バックトラッキング＋`[[SILENCE:1000]]` |

**クロージングの型（台本のバリエーション。分析結果に応じて選ばせる）**
1. **沈黙クロージング型**：選択/仮定型の質問→直後に2秒沈黙。
2. **言い切り型**：「どうですか？」を封印し「これが一番合っています。進めましょう。」と断言。
3. **共感→畳みかけ型(LAER)**：異議を受け止め→繰り返し→「だからこそ今」と語尾を下げて締める。
4. **真誠(Sincerity)型**：最終局面で低く遅く「正直に言わせていただきます…」。
5. **価格後サイレンス型**：金額提示→3〜5秒沈黙。

**鉄則（最優先3つ）**：①断言の語尾は必ず下げる ②クロージング質問・価格の直後は黙る ③最後ほど低く・ゆっくり。
（出典等の詳細根拠は別途リサーチレポート参照。ここには実装に必要な結論のみ記載）

## 4. 独自記法 `[[SILENCE:ms]]`（このシステムの内部仕様）

- 台本テキスト内に `[[SILENCE:2000]]` と書くと「ここに2000ミリ秒の無音」を意味する。
- これは **Fishには渡さない**。サーバー側で分割マーカーとして使い、ffmpegで無音を挿入する。
- `[pause]` / `[long pause]` など**角括弧1個のタグはFishにそのまま渡す**（Fishが解釈する）。`[[二重括弧]]`だけが無音マーカー。

### A. 台本生成プロンプトの刷新 — `src/closing.ts`
- `buildIdealClosingPrompt()`（≈L26-58）と `buildSectionPrompt()`（≈L143）に、**§3のデリバリー早見表と型**を反映。
  - 営業ターンに `[pause]` / `[低い声で]` 等のインラインタグと、`[[SILENCE:ms]]` を適切に埋め込ませる。
  - 必ず**客との対話**（テストクロージング→客の反応→受け止め→言い切り）を含める。
  - クロージング質問・価格提示の直後には必ず `[[SILENCE:...]]` を置く。
  - 既存の「分析の弱点を反映」(analysisFindings) は維持。

### B. 2モード対応（対話版／語り版）
- `generateIdealClosingScript()`（≈L82）に `mode: 'dialogue' | 'monologue'` を追加。
  - **dialogue**：従来通り 営業:/客: の掛け合い台本。
  - **monologue**：営業のターンのみ。**客のターンは本文から落とし、その尺ぶんを `[[SILENCE:1500]]` 程度の間に置換**（＝相手が話している想定の沈黙）。
- `parseClosingDialogue()`（≈L64）：`営業:`/`客:` 分解に加え、各ターン内の `[[SILENCE:ms]]` を「無音セグメント」要素として分離するように拡張。

### C. 無音挿入つき合成 — `src/closing.ts` `synthesizeDialogue()`（≈L413）＋ `concatAudio()`
- ターン列を走査し、テキストは `provider.synthesize()` で音声化、`[[SILENCE:ms]]` は**無音audioセグメント**を生成して列に挿入。
- 無音生成は ffmpeg（`anullsrc=r=44100:cl=mono` を ms 指定で生成）。既存の `concatAudio()` の連結列に混ぜる。
- monologueモードでは客ターンが無音に置換されているので、同じ経路で自然に動く。

### D. Fish呼び出しパラメータ — `src/voice.ts` `FishAudioProvider.synthesize()`（≈L75-92）
- ヘッダーに `model: 's2.1-pro'` を追加。
- payload に `temperature: 0.88, top_p: 0.85, prosody: { speed: 0.92 }` を追加。
- **`emotion` は追加しない**（存在しないため）。
- `reference_id`（本人 / 客=汎用声）の現仕様は維持。
- 既存の `mp3_bitrate/normalize/latency` は維持。

### E. エンドポイント — `src/server.ts`
- `/api/voice/generate-sample`（≈L635）と `/api/voice/say-line`（≈L722）に `mode`（dialogue/monologue）を受け取る引数を追加し、closing.ts へ渡す。
- `FEATURE_VOICE_CLONE` の403ガードは維持。キャッシュキーに `mode` を含める（モード別にキャッシュ）。

### F. UI：声設定を評価の上へ＋モード選択 — `public/index.html`
