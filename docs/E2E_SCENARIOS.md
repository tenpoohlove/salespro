# E2Eシナリオ — p3 声クローン添削
版 1.0 / 2026-06-12 / 各シナリオ: 前提/操作/期待/データ/異常/合否/トレース

## 正常系
### E2E-01 商談アップ→添削（評価軸）
- 前提: ログイン済・Anthropic/OpenAIキー設定済
- 操作: 商談音声アップ→文字起こし→分析実行
- 期待: 商談クロージング評価軸(Pain/BANT/MEDDIC等)＋非言語観点補足のレポートがSSEで表示
- データ: サンプル商談mp3(短)
- 合否: 商談向け評価軸セクションと非言語補足が出る / トレース: FR-DATA-001/010/012

### E2E-02 理想クロージング声見本の生成
- 前提: E2E-01完了・Fish Audioキー設定済・本人同意ON
- 操作: 「理想クロージング音声見本」実行
- 期待: 理想台本生成→本人声クローン→mp3がプレイヤーで再生可能
- 合否: 音声が返り再生できる(Mock環境ではダミー音声) / トレース: FR-DATA-011, FR-VOICE-001/002/003

### E2E-03 キャッシュで2回目0円
- 操作: 同じ商談で声見本を再生成
- 期待: audio_cacheヒットでFish Audio未呼出・即返却
- 合否: API呼出0回 / トレース: FR-VOICE-004

## 異常系
### E2E-04 APIキー未設定
- 操作: キー無しで分析/声見本
- 期待: 「設定ページでキーを入力」と日本語エラー。サーバーキーで黙って補完しない
- 合否: エラー表示＋サーバー非フォールバック / トレース: SEC-001, FR-USR-001/002

### E2E-05 複数話者で本人特定不可
- 操作: 複数話者商談で声見本生成
- 期待: 本人の区間指定UIを促す
- 合否: 区間指定を要求 / トレース: FR-VOICE-001, RISK-005

## 境界系
### E2E-06 同意なしは生成拒否
- 操作: 本人同意OFFのまま声見本生成
- 期待: 生成を拒否し同意を求める(なりすまし防止)
- 合否: 生成されない / トレース: SEC-002, RISK-003
